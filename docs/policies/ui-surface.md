# UI Surface Policy

> Store 앱의 운영형 페이지는 카드보다 레이아웃, 타이포, 구분선으로 위계를 만든다.

## 목적

- 페이지 전체가 하나의 큰 카드처럼 보이는 인상을 줄인다.
- 주문, 문의, 내 정보 같은 화면을 동일한 박스 템플릿이 아니라 목적 기반 구조로 읽히게 만든다.
- 데스크톱과 모바일 모두에서 여백과 스캔성이 먼저 보이도록 한다.

## 기본 원칙

1. 페이지 전체를 감싸는 카드 사용을 금지한다.
2. 섹션 기본 구조는 `제목 + 설명 + row list + divider`로 만든다.
3. 카드나 패널은 독립된 상태 블록 또는 경고 블록에만 사용한다.
4. 그림자보다 spacing, alignment, border, background contrast로 위계를 만든다.
5. 동일한 radius, border, shadow를 모든 영역에 반복 적용하지 않는다.

## 카드 허용 기준

다음 경우에만 카드 또는 패널을 허용한다.

- 결제 요약, 상태 요약처럼 독립적으로 읽혀야 하는 정보 묶음
- 회원탈퇴, 환불 경고처럼 위험도가 분명한 액션 블록
- 프로모션, 배너, 광고처럼 메인 작업 흐름과 분리되어야 하는 보조 영역
- 모바일에서 터치 단위가 명확해야 하는 단일 액션 모듈

## 카드 금지 기준

다음 경우에는 카드 대신 레이아웃을 사용한다.

- 페이지 전체 외곽
- 단순 메뉴 목록
- 섹션 구획 자체
- 본문 리스트를 감싸는 용도
- 데스크톱에서 빈 공간을 채우기 위한 용도

## 권장 구조

운영형 페이지는 아래 4개 블록으로 조합한다.

- `PageIntro`: eyebrow, title, description, optional actions
- `PageSection`: section title, short utility copy, row list
- `PageLinkRow`: 좌측 라벨과 설명, 우측 상태 또는 meta
- `PageAsidePanel`: 요약 정보, 경고, 프로모션, 빠른 설정

## 시각 규칙

- 배경은 열린 surface로 유지하고 불필요한 외곽선은 줄인다.
- 본문 위계는 heading 크기, 상하 spacing, divider로 만든다.
- hover는 배경색 한 단계 변화만 사용한다.
- accent color는 1개만 사용하고, 기본 텍스트는 짙은 무채색 위주로 유지한다.
- 그림자는 필요한 패널에만 아주 약하게 사용하거나 생략한다.

## 마이페이지 적용 기준

- 상단: 사용자 이름, 연락처, 현재 상태
- 본문 좌측: 주문, 지원, 설정
- 본문 우측: 상태 요약, 최근 동작, 프로모션
- 하단: 회원탈퇴 같은 위험 액션

페이지를 구성할 때 "카드를 제거해도 구조가 유지되는가"를 먼저 확인한다. 유지되지 않으면 카드가 아니라 레이아웃부터 다시 설계한다.
