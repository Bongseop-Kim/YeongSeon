## 프로세스

### Sale
1. 배송중 상태일 때 취소 불가능하도록 변경

### Repair
1. 수선중 상태일 때 취소 불가능하도록 변경

### Custom Order
1. 샘플 상태 추가 (샘플 원단 제작중, 샘플 원단 배송중, 샘플 봉제 제작중, 샘플 넥타이 배송중, 샘플 배송완료, 샘플 승인)
   - 상태 전이: 접수 → 샘플 원단 제작중 → 샘플 원단 배송중 → 샘플 봉제 제작중 → 샘플 넥타이 배송중 → 샘플 배송완료 → 샘플 승인 → 제작중 → ...
   - 반려/재작업은 시스템 밖에서 관리자 직접 처리
   - 취소는 기존 클레임(cancel)으로 처리 (단, 부분 환불 필요 — 항목 3 참조)
2. 제작중 상태일 때 취소 불가능하도록 변경
3. 샘플 비용 별도 저장 및 부분 환불 처리
   - `orders` 테이블에 `sample_cost` 컬럼 추가
   - `create_custom_order_txn`에서 샘플 비용 분리 저장
   - `calculate_custom_order_amounts`에서 샘플 비용 별도 반환
   - 샘플 단계 취소 시 환불액 = 결제 금액 - sample_cost (일반 취소 전액 환불과 분기 필요)
4. 상호배타적 옵션 `dimple`, `spoderato`, `fold7` 중 다중 선택 가능하도록 변경

### Quote Request
1. 관리자 화면과 구매자 주문 목록 화면에서 custom-order와 quote-request가 구분되도록 점검 필요
   - 관리자: 커스텀 주문과 견적 요청이 나눠져 있지 않을 가능성 높음
   - 구매자: 확인 필요

### Design (AI 디자인 생성)
1. 토큰 환불 정책 수립 필요
   - 환불 신청 이후부터는 토큰 사용 불가 처리 필요
   - 보너스 토큰 개념 도입 검토 (많이 살수록 싸게가 아닌 보너스 지급 방식)
   - 보너스 토큰은 환불 불가 → 비싸게 사놓고 남은 토큰 환불받는 사례 방지 + 환불 로직 단순화
2. 고객이 토큰을 환불할 수 있는 기능 위치 검토 필요
