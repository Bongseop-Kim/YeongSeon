## 프로세스

### Sale

1. 배송중 상태일 때 취소 불가능하도록 변경

### Repair

1. 수선중 상태일 때 취소 불가능하도록 변경
2. 비용 값이 public.custom_order_pricing_constants 테이블에서 관리되고 있는지 확인 필요 ('REFORM_BASE_COST', 'REFORM_SHIPPING_COST 키)

### Custom Order

1. 샘플 상태 추가 (샘플 원단 제작중, 샘플 원단 배송중, 샘플 봉제 제작중, 샘플 넥타이 배송중, 샘플 배송완료, 샘플 승인)
   - 상태 전이: 접수 → 샘플 원단 제작중 → 샘플 원단 배송중 → 샘플 봉제 제작중 → 샘플 넥타이 배송중 → 샘플 배송완료 → 샘플 승인 → 제작중 → ...
   - 반려/재작업은 시스템 밖에서 관리자 직접 처리
   - 취소는 기존 클레임(cancel)으로 처리 (단, 부분 환불 필요 — 항목 3 참조)
2. 제작중 상태일 때 취소 불가능하도록 변경
3. 샘플 비용 별도 저장 및 부분 환불 처리
   - `orders` 테이블에 `sample_cost` 컬럼 추가
   - `create_custom_order_txn`에서 샘플 비용 분리 저장
   - `calculate_custom_order_amounts`에서 샘플 비용 별도 반환
   - 샘플 단계 취소 시 환불액 = 결제 금액 - sample_cost (일반 취소 전액 환불과 분기 필요)
4. 상호배타적 옵션 `dimple`, `spoderato`, `fold7` 중 다중 선택 가능하도록 변경
5. 주문 요청 접수 성공 시 고객에게 카카오톡 알림 발송 (접수 완료 안내)
6. 메시지 발송 시나리오 (카카오톡 기본, 실패 시 SOLAPI SMS 폴백)
   - 샘플 있는 경우: 주문 접수 → 샘플 비용 요청 카카오톡 발송 → 입금 확인 후 관리자 승인 → 샘플 제작 진행 → 샘플 발송과 동시에 본 견적 비용 카카오톡 발송 → 입금 확인 후 관리자가 제작중으로 상태 변경
   - 샘플 없는 경우: 주문 접수 → 관리자 승인 → 견적 비용 카카오톡 발송 → 입금 확인 후 관리자가 제작중으로 상태 변경
7. SMS 폴백 조건: 카카오톡 미가입 또는 메시지 전송 오류인 경우 SOLAPI SMS 발송
   - SMS 발송 전 OTP 본인 소유 폰 인증 필수
   - 인증 성공 시 계정에 폰 번호 등록 + OTP 확인 여부 기록
8. 개인정보 수집 동의 및 마케팅 수신 동의 관리 기능 추가 필요
9. admin 주문제작 상세 페이지에서 "함께 결제된 주문" 섹션 제거 (주문제작은 단독 결제만 존재)

### Quote Request

1. 관리자 화면과 구매자 주문 목록 화면에서 custom-order와 quote-request가 구분되도록 점검 필요
   - 관리자: 커스텀 주문과 견적 요청이 나눠져 있지 않을 가능성 높음
   - 구매자: 확인 필요

### Design (AI 디자인 생성)

1. 토큰 환불 정책 수립 필요
   - 환불 신청 이후부터는 토큰 사용 불가 처리 필요
   - 보너스 토큰 개념 도입 검토 (많이 살수록 싸게가 아닌 보너스 지급 방식)
   - 보너스 토큰은 환불 불가 → 비싸게 사놓고 남은 토큰 환불받는 사례 방지 + 환불 로직 단순화
2. 고객이 토큰을 환불할 수 있는 기능 위치 검토 필요

## 정책

### Payment

1. Toss Payments MCP 연결하여 리팩토링 검토

### Coupon

1. percentage, fixed 2개 타입으로 실제 구현되어 있는지 확인 필요

### Token

1. 디자인 토큰에 포인트 정책 도입 검토
2. 기존 주문 프로세스와 별도로 관리할 필요가 있는지 검토

### Point

1. 유저 포인트 확인 화면 구현 필요 (store — 마이페이지 내 포인트 내역)
2. 주문폼(Order / order-form)에서 보유 포인트 표시 및 사용 UI 구현 필요
   - 최소 1,000원 이상부터 100원 단위로 사용 가능
3. admin 고객 관리에서 유저 포인트 조회·수동 지급·차감 기능 추가 필요
4. 포인트 사용·적립·차감 이력 관리 기능 필요

### Notification

1. 알림 채널 우선순위: 카카오톡 기본, 실패(미가입/오류) 시 SOLAPI SMS 폴백
2. SMS 폴백 사전 조건: OTP 본인 소유 폰 인증 후 계정에 번호 등록, 인증 여부 기록
3. 개인정보 수집 동의 및 마케팅 수신 동의 관리 테이블/API 설계 필요
4. 발송 이력(수신자, 채널, 내용, 성공/실패) 저장 필요
