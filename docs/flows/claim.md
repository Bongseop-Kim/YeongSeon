# Claim 플로우 (취소 / 반품 / 교환)

> 진입점: store /order/:orderId → 취소/반품/교환 요청 버튼 → /order/claim/:type/:orderId/:itemId

## 상태 전이

| 클레임 유형 | 현재 상태   | 다음 상태 | 방향   | 조건        |
| ----------- | ----------- | --------- | ------ | ----------- |
| 취소        | 접수        | 처리중    | 순방향 | admin 처리  |
| 취소        | 처리중      | 완료      | 순방향 | admin 처리  |
| 취소        | 처리중      | 접수      | 롤백   | memo 필수   |
| 반품        | 접수        | 수거요청  | 순방향 | admin 처리  |
| 반품        | 수거요청    | 수거완료  | 순방향 | admin 처리  |
| 반품        | 수거완료    | 완료      | 순방향 | admin 처리  |
| 반품        | 수거요청    | 접수      | 롤백   | memo 필수   |
| 교환        | 접수        | 수거요청  | 순방향 | admin 처리  |
| 교환        | 수거요청    | 수거완료  | 순방향 | admin 처리  |
| 교환        | 수거완료    | 재발송    | 순방향 | admin 처리  |
| 교환        | 재발송      | 완료      | 순방향 | admin 처리  |
| 교환        | 수거요청    | 접수      | 롤백   | memo 필수   |
| 모든 유형   | 접수~재발송 | 거부      | 거부   | admin 처리  |
| 모든 유형   | 거부        | 접수      | 복원   | 오거부 복원 |

## 비즈니스 규칙

- 수거완료, 재발송, 완료 상태에서는 롤백 불가
- 롤백 시 memo 필수
- 거부 → 접수 복원은 항상 가능 (오거부 복원)
- 반품/교환은 배송중, 배송완료 상태 주문에서만 신청 가능 (완료 상태에서는 불가)
- 취소는 주문 유형별 가능 상태가 다름 (sale: 대기중~진행중 / repair: 대기중, 발송대기·발송중(미구현) / custom: 대기중, 접수, 제작중, 샘플 단계(미구현))

## 화면

| 앱    | 화면        | 경로                                |
| ----- | ----------- | ----------------------------------- |
| store | 클레임 신청 | /order/claim/:type/:orderId/:itemId |

| admin | 클레임 목록 | /claims                             |
