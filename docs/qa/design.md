# Design QA 시나리오

> 토큰 구매/환불과 AI 생성 두 흐름을 모두 커버합니다.
> E2E 테스트 자동 생성의 입력으로 사용됩니다.

## 생성 라우팅 골든셋

| 프롬프트                                                                                         | 기대 라우팅  |
| ------------------------------------------------------------------------------------------------ | ------------ |
| 첨부한 이미지를 올 패턴으로 넥타이 디자인해줘                                                    | `fal_tiling` |
| 첨부한 이미지를 우측 하단에 위치 시켜줘                                                          | `fal_edit`   |
| 첨부한 이미지의 올 패턴으로 뿌리고 넥타이 원단 배경 색은 녹색으로 해줘                           | `fal_tiling` |
| 첨부한 이미지 참고해서 비슷하게 만들어줘                                                         | `openai`     |
| 스트라이프 배경에 작은 새 올 패턴 넥타이 디자인해줘                                              | `openai`     |
| 포인트 위치가 너무 높아 아래로 내려줘 50px 정도 아니면 적당히 내려봐                             | `fal_edit`   |
| 패턴의 사이즈가 너무 커 50% 정도 사이즈로 줄여줘 한 줄에 패턴이 20개 정도 들어올 정도의 사이즈로 | `fal_edit`   |
| 포인트에 위치한 내 로고가 첨부한 이미지랑 일치하지가 않아. 다시 수정해줘                         | `fal_edit`   |

## 수동 라우팅 확인

1. CI 이미지를 첨부하고 `올 패턴` 요청 시 `fal_tiling`으로 라우팅되는지 확인한다.
2. 생성 결과가 있는 상태에서 `아래로 내려줘` 요청 시 `fal_edit`로 라우팅되는지 확인한다.
3. `참고해서 비슷하게` 요청 시 `openai`로 유지되는지 확인한다.
4. base image 없이 수정 요청 시 생성 호출 대신 UI 안내 메시지로 차단되는지 확인한다.

### SC-design-017: 토큰 결제 실패 후 주문 상태 복구

- **참조 규칙**: PR-payment-005
- **우선순위**: critical

| 단계  | 내용                                                        |
| ----- | ----------------------------------------------------------- |
| Given | store 토큰 구매 페이지에서 결제를 시도한 상태               |
| When  | Toss 결제가 실패하여 /token-purchase/fail 페이지로 이동한다 |
| Then  | 토큰 잔액 변화가 없고 토큰 주문 상태가 대기중으로 복구된다  |

**경로**: `store /token-purchase/fail`
**결과**: untested
**이슈**: `/token-purchase/fail` 페이지는 실패 결과를 표시하고 복구 완료 상태를 확인하는 역할만 하며, 실제 상태 변경(토큰 잠금 해제·주문 상태 복원)은 `confirm-payment` Edge Function이 Toss 실패 처리 시 자체적으로 수행한다. fail 페이지에서 별도 unlock 호출 불필요. SC-design-002와 구분: SC-design-002는 결제 실패 자체의 잔액 무변화를 검증하고, 이 시나리오는 `/token-purchase/fail` 페이지 진입 후 `confirm-payment`가 완료한 복구 결과(잔액·주문 상태)가 올바르게 반영되어 있는지를 검증함
