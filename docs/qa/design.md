### SC-design-017: 토큰 결제 실패 후 주문 상태 복구

- **참조 규칙**: PR-payment-005
- **우선순위**: critical

| 단계  | 내용                                                        |
| ----- | ----------------------------------------------------------- |
| Given | store 토큰 구매 페이지에서 결제를 시도한 상태               |
| When  | Toss 결제가 실패하여 /token-purchase/fail 페이지로 이동한다 |
| Then  | 토큰 잔액 변화가 없고 토큰 주문 상태가 대기중으로 복구된다  |

**경로**: `store /token-purchase/fail`
**결과**: untested
**이슈**: confirm-payment Edge Function에서 Toss 실패 시 unlock을 자체 처리하므로 fail 페이지에서 별도 호출 불필요 (정상). SC-design-002와 구분: 이 시나리오는 `/token-purchase/fail` 페이지의 복구 동작(잔액·주문 상태 복원)을 검증하며, SC-design-002는 결제 실패 자체의 잔액 무변화를 검증함
